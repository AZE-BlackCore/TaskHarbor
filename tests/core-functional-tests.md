## 测试范围

### 1. 数据库层优化测试 (Phase 1)

#### 1.1 延迟保存机制
**测试项**: 验证数据库延迟保存是否正常工作

**测试步骤**:
1. 启动应用
2. 快速连续创建 5 个任务
3. 观察控制台日志
4. 检查数据库文件写入时间

**预期结果**:
- 控制台显示 "Database saved at [timestamp]"
- 1 秒内多次操作只保存一次
- 性能提升明显

**测试结果**: ⏳ 待测试

#### 1.2 事务支持
**测试项**: 验证事务回滚功能

**测试步骤**:
1. 使用 `runTransaction` 批量创建任务和子任务
2. 故意制造错误（如引用不存在的项目）
3. 验证数据是否回滚

**预期结果**:
- 事务失败时所有操作回滚
- 数据库保持一致性

**测试结果**: ⏳ 待测试

#### 1.3 数据验证触发器
**测试项**: 验证触发器是否阻止无效数据

**测试步骤**:
1. 尝试创建任务，使用不存在的项目 ID
2. 尝试设置进度为 150%
3. 观察错误提示

**预期结果**:
- 抛出 "Project not found" 错误
- 抛出 "Progress must be between 0 and 100" 错误

**测试结果**: ⏳ 待测试

#### 1.4 ID 和时间戳统一
**测试项**: 验证 ID 格式和时间戳格式

**测试步骤**:
1. 创建任务
2. 检查返回的 ID 格式
3. 检查 createdAt 和 updatedAt 格式

**预期结果**:
- ID 格式：`task_xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`
- 时间戳格式：`2026-03-30T12:34:56.789Z` (ISO 8601)

**测试结果**: ⏳ 待测试
