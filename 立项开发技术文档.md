## 一、项目概述

### 1.1 项目背景

在现代工作和学习环境中，任务管理已成为个人和团队提高效率的核心需求。现有工具要么过于简单（便签、待办清单），要么过于复杂（Jira、Project），缺少一款既轻量又功能完整、既适合个人使用又支持项目化管理的桌面应用。

**Todo List Desktop** 是一款基于 Electron 技术栈的跨平台桌面任务管理软件，提供：
- 🎯 项目化任务管理（个人/公司项目）
- 📊 五种视图展示（列表、甘特图、看板、日历、仪表盘）
- ⏱️ 时间追踪与工时统计
- 🏷️ 灵活的标签系统
- 📤 多格式导出（Excel/CSV/Markdown）
- 💻 原生桌面体验（悬浮窗口、系统通知）

### 1.2 项目目标

#### 核心目标
1. **功能完整性** - 覆盖任务管理全生命周期
2. **用户体验** - 流畅交互、美观界面、响应式设计
3. **性能优秀** - 启动<2 秒、60 FPS 滚动、即时响应
4. **跨平台** - Windows、macOS、Linux 全支持
5. **数据安全** - 本地 SQLite 数据库，无需联网

#### 成功指标
- 用户 5 分钟内创建第一个项目和任务
- 视图切换时间 < 200ms
- 支持 1000+ 任务无卡顿
- 打包体积 < 100MB

### 1.3 适用范围

**目标用户**：个人任务管理、项目进度跟踪、时间记录、学生作业管理

**使用场景**：个人日常任务、项目跟踪、工时统计、团队协作（未来）

### 3.2 数据库设计

**核心表结构**：

1. **projects** - 项目表
   - id, name, type(personal/company), description, color, createdAt, updatedAt

2. **tasks** - 任务表（16 字段）
   - id, projectId, moduleId, module, functionModule
   - description, progress, status, assignee
   - startDate, estimatedEndDate, actualEndDate
   - issues, notes, createdAt, updatedAt

3. **subtasks** - 子任务表
   - id, taskId, description, completed, orderIndex, createdAt

4. **timelogs** - 时间日志表
   - id, taskId, startTime, endTime, durationSeconds, description, createdAt

5. **task_dependencies** - 任务依赖表
   - id, taskId, dependencyTaskId, type, createdAt

6. **task_tags** - 任务标签表
   - id, taskId, tagName, tagColor, createdAt

**索引优化**：
- tasks: projectId, status, assignee, startDate
- subtasks: taskId
- timelogs: taskId
- dependencies: taskId
- tags: taskId

## 五、技术规范

### 5.1 代码规范

**TypeScript**：
- 启用严格模式
- 禁止使用 `any`
- 所有 API 有类型定义
- 接口定义对象类型

**组件规范**：
- 函数组件 + Hooks
- 组件名 PascalCase
- Props 使用 interface
- 单一职责

**样式规范**：
- 优先 Tailwind CSS
- 禁止内联样式
- CSS 变量支持主题

**命名规范**：
```typescript
// 文件
TaskList.tsx      // 组件 PascalCase
task-store.ts     // Store kebab-case

// 变量
const taskList    // camelCase
const MAX_COUNT   // UPPER_SNAKE_CASE
interface Task    // PascalCase
```

### 5.2 Git 规范

**Commit Message**（Conventional Commits）：
```
feat(task): 添加子任务管理功能

- 实现子任务 CRUD
- 添加进度统计

Closes #123
```

**分支管理**：
```
main          - 主分支
develop       - 开发分支
feature/*     - 功能分支
bugfix/*      - Bug 修复
release/*     - 发布分支
```

### 5.3 测试规范

**测试类型**：
- 单元测试：Jest + React Testing Library
- 集成测试：IPC、数据库
- E2E 测试：Playwright（规划）

**覆盖率要求**：
- 语句覆盖率 > 80%
- 分支覆盖率 > 70%
